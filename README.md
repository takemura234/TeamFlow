# TeamFlow 開発プロジェクト仕様書

**チームリーダー支援 タスク管理・スケジュール管理アプリ**

| 項目 | 内容 |
|------|------|
| プロジェクト名 | TeamFlow（仮称） |
| 対象 | スギヤス株式会社 自動車整備用リフト開発チーム |
| 作成日 | 2026年6月13日 |
| バージョン | 1.1（完全無料プラン対応） |
| 担当TL | 武村TL |
| チーム規模 | TL × 1名 ＋ メンバー × 6名（計7名） |

---

## 1. プロジェクト概要

### 1.1 背景・課題

現状、開発チームでは以下の問題が発生している。

- スケジュールを一度策定しても、遅延が発生した際に軌道修正が行われない
- TODOリストの入力が煩雑で三日坊主になりやすい
- 上司・チームリーダーへの遅延アラームが機能していない
- メンバー間の進捗把握と優先順位共有が属人化している

### 1.2 目的

- スケジュール進行の円滑化
- やることの忘れ防止
- 優先順位の把握と共有
- 遅延発生時の早期検知と上司への自動通知

### 1.3 アプリ名（仮）

**TeamFlow** ― チームの流れを止めない、を合言葉にしたプロジェクト管理ツール

## 2. 対象ユーザーと権限

| ロール | 対象者 | 主な操作 |
|--------|--------|----------|
| 管理者 (Admin) | 武村TL（チームリーダー） | 全タスク作成・編集・削除、メンバー管理、アラーム設定、AIレポート閲覧 |
| メンバー (Member) | 開発メンバー6名 | 自分のタスク入力・進捗更新、音声入力、コメント追加 |
| 上司 (Viewer) | 武村TLの上長 | ダッシュボード閲覧、アラーム受信（LINE通知）、コメント可 |

## 3. 機能要件

### 3.1 タスク管理機能

#### 3.1.1 タスク登録

- タスク名、担当者、開始日、期限日、優先度（高/中/低）、進捗（%）を登録
- サブタスク（チェックリスト）を最大10件まで追加可能
- 関連開発テーマとのひも付け
- コメント・メモ欄（テキスト）

#### 3.1.2 音声入力（スマホ対応）

- スマホブラウザのWeb Speech API（iOS Safari / Android Chrome対応）を利用
- 「タスク名」「期限」「担当者」「優先度」をセリフ形式で話しかけて入力
- 例：「来週金曜までに横井さんに油圧設計レビューをお願い、優先度高」
- 入力後に確認画面を表示し、修正後に保存

#### 3.1.3 進捗更新

- メンバーは自分のタスクの進捗をスライダー（0%〜100%）またはステータス選択で更新
- ステータス選択肢：未着手 / 進行中 / レビュー待ち / 完了 / 保留
- 更新時にコメントを添付可能

### 3.2 スケジュール管理機能

#### 3.2.1 ガントチャート表示

- 開発テーマごとのガントチャートをPC・スマホで閲覧
- 当日ラインを表示し、遅延タスクを赤色でハイライト
- TLはブラウザ上でドラッグ操作により期限を変更可能（軌道修正）

#### 3.2.2 マイルストーン設定

- 各開発テーマに対してマイルストーンを設定（例：設計完了、試作完了、出荷）
- マイルストーン到達状況を進捗ダッシュボードに表示

### 3.3 アラーム・通知機能

#### 3.3.1 通知トリガー（自動検知）

| トリガー条件 | 通知先 | 通知内容 |
|-------------|--------|----------|
| 期限の3日前でも進捗50%未満 | 担当メンバー + TL | 「〇〇タスクが遅延リスクあり。現在進捗XX%」 |
| 期限超過（当日23:59時点） | TL + 上司 | 「〇〇タスクが期限超過。担当：〇〇」 |
| 1週間進捗更新がないタスク | 担当メンバー | 「〇〇タスクの進捗を更新してください」 |
| マイルストーン達成 | TL + チーム全員 | 「マイルストーン〇〇を達成しました！」 |
| 緊急タスク（優先度：高）追加 | チーム全員 | 「緊急タスクが追加されました：〇〇」 |

#### 3.3.2 通知チャネル

- LINE通知（LINE Messaging API 使用）：メンバー・TL・上司のLINEアカウントに直接送信
- アプリ内通知（ベルマーク）：未読バッジ表示
- 通知の有効/無効はユーザー設定で個別に切り替え可能

### 3.4 AI支援機能（Google Gemini API使用）

#### 3.4.1 遅延時の自動再提案

- タスク遅延検知時に、AIがタスクの内容・残工数・チームリソースを分析して軌道修正案を提示
- 提示例：「このタスクは3日遅延しています。横井さんの今週リソースが空いているため、サブタスクAを分担することを提案します」
- TLが承認すればタスクの担当・期限を自動更新

#### 3.4.2 チャット形式AIアドバイザー

- アプリ内チャット欄でAIに相談できる
- 例：「今週のチームで一番リスクの高いタスクはどれ？」「優先度の高いタスクを整理して」
- チームの現在のタスク状況をコンテキストとして渡すことで、的確なアドバイスを返す
- AIはGoogle Gemini APIを使用（gemini-2.5-flash・無料枠）

#### 3.4.3 週次サマリーレポート

- 毎週月曜朝8時にAIが週次レポートを自動生成
- 内容：先週の完了タスク、今週の重要タスク、遅延リスク、AIコメント
- TLと上司にLINEおよびアプリ内通知で配信

## 5. 技術スタック

| レイヤー | 採用技術 | 選定理由 |

## 11. LINE通知連携フロー

### 11.1 セットアップ手順

1. LINE Developersでチャンネル（Messaging API）を作成
2. チャンネルアクセストークン（長期）を取得しRender環境変数に設定
3. 各ユーザーのLINE User IDをアプリ設定画面から登録（QRコード友達追加フロー）
4. 通知はPush Message APIで個別送信（1:1）またはグループ送信に対応

### 11.2 コスト

- LINE Messaging API：月200通まで無料（コミュニケーションプラン）
- 7名チームの通常運用（月70〜100通程度）であれば無料枠内で運用可能
- 緊急通知（全員7名宛）が頻発すると超過する可能性あり。通知頻度を設計で抑制する

## 12. 制約・リスク・考慮事項

| 項目 | 内容 | 対策 |
|------|------|------|
| Render無料プラン スリープ | 15分無操作でスリープ（起動に50秒程度） | UptimeRobot（無料）で5分ごとにPingを送り、スリープを防止する |
| LINE通知 月200通上限 | コミュニケーションプランは月200通まで（超過不可・追加購入不可） | 通知をまとめて1通に集約する設計にし、不要な全員通知を減らす。超過時はライトプラン（月5,000円）への移行を検討 |
| Gemini 無料枠 データ利用規約 | 無料枠利用時、送信したプロンプトがGoogleのモデル学習に使われる可能性あり | AIに送る情報は「タスク名・進捗・日付」に限定し、個人名・製品詳細等の機密情報を含めないプロンプト設計にする |
| Gemini 無料枠 レート制限 | 1分15リクエスト（RPM）の制限あり | 7名チームでの同時アクセスは現実的に少ないため問題なし。万一の429エラーにはリトライ処理を実装 |
| LINE User ID取得 | メンバー全員がLINEアカウントでボットを友達追加し通知許可が必要 | 初回セットアップ手順書を作成し、TLが展開 |
| 音声入力精度 | Web Speech APIは社内ネット環境・方言によって認識精度が変わる | 入力後に確認・修正ステップを必ず挟む |
| iOS音声入力 | Web Speech APIはiOS Safariで動作するが精度・安定性に差異あり | フォールバックとして手入力モード常時提供 |
| データ移行 | 現状エクセル等で管理しているタスクのインポート機能 | Phase 4でCSVインポート機能を検討 |

## 13. 成功指標（KPI）

- **タスク入力率**：チームメンバー全員が週3回以上アプリにアクセス・更新している

- **上司報告工数**：手動での進捗報告メールが月1回以下になる
- **TL満足度**：武村TLが「スケジュール軌道修正が楽になった」と評価する

## アプリ内通知

ログイン時に次の条件を自動確認し、対象ユーザーの通知欄へ1日1回まで表示します。

- 期限まで3日以内かつ進捗50%未満: 担当者と武村TL
- 期限超過: 武村TL
- 7日以上更新がないタスク: 担当者
- 高優先度タスク追加: アプリ内通知を有効にしている全員
- マイルストーン達成: アプリ内通知を有効にしている全員

管理者は通知設定画面の「遅延リスクを今すぐ確認」から手動実行もできます。

## 複数プロジェクト管理

管理者は管理画面からプロジェクトを何件でも登録できます。

- プロジェクト名、期間、説明を登録・編集
- タスク登録時に所属プロジェクトを選択
- マイルストーン登録時に所属プロジェクトを選択
- ダッシュボード、タスク一覧、ガントチャートをプロジェクト別に絞り込み
- タスクまたはマイルストーンが存在するプロジェクトは誤削除防止のため削除不可

## ガントチャート編集

管理者はガントチャート上でタスク期間を変更できます。

- バー中央をドラッグ: 開始日と期限を同じ日数だけ移動
- 左端をドラッグ: 開始日を変更
- 右端をドラッグ: 期限を変更
- 今日の日付をオレンジ色の縦線で表示
- 変更内容はタスクコメントへ自動記録

## 追加された運用機能

- 音声入力: 新規タスク画面を開くと「音声で入力」ボタンを表示
- パスワード変更: 通知設定画面から本人が変更
- LINE連携: LINE User IDを保存し、テスト通知を送信
- AI再計画: 遅延リスク欄から担当者・期限案を生成し、承認後に反映
- 週次レポート: AI画面から手動生成、または毎週月曜8時に自動生成
- 日次通知チェック: 毎日8時5分に自動実行
- SQLiteバックアップ: `python backup_db.py` で `backups/` に保存
- 管理者バックアップ: 管理画面の「DBバックアップを作成」から作成・ダウンロード
- メンバーパスワード再設定: 管理画面のメンバー一覧から一時パスワードを設定
- 自動テスト: `python -m unittest discover -s tests -v`
- CSV移行: 管理画面からタスクをUTF-8 CSVで書き出し・一括読み込み

CSVの列は `title, project, assignee, start_date, due_date, priority, status, progress, description` です。`assignee` はメンバー名またはログインIDを指定できます。日本語の列名にも対応しています。

スマホではHTTPSで公開したTeamFlowをブラウザで開き、「ホーム画面に追加」するとPWAとして起動できます。画面資産のみをキャッシュし、ログイン情報やAPIデータはキャッシュしません。

### 本番環境変数

- `TEAMFLOW_SECRET_KEY`: セッション署名用の十分に長いランダム値
- `TEAMFLOW_DB`: DBファイルパス。Renderでは `/var/data/teamflow.db`
- `TEAMFLOW_BACKUP_DIR`: バックアップ保存先。Renderでは `/var/data/backups`
- `TEAMFLOW_COOKIE_SECURE=1`: HTTPS限定Cookie
- `TEAMFLOW_ENABLE_SCHEDULER=1`: 日次通知・週次レポートを有効化
- `GEMINI_API_KEY`: Gemini機能を有効化
- `LINE_CHANNEL_ACCESS_TOKEN`: LINE Push Messageを有効化

`render.yaml` は無料Webサービス用です。SQLiteは一時領域に保存されるため、再起動や再デプロイでデータが初期化される可能性があります。管理画面から定期的にDBバックアップをダウンロードしてください。有料の永続ディスク構成は `render-paid.yaml` に残しています。

## Render本番公開

[Renderへデプロイ](https://render.com/deploy?repo=https://github.com/takemura234/TeamFlow)

RenderのBlueprint作成画面で、次の秘密値を入力します。値はGitHubへ保存されません。

- `TEAMFLOW_INITIAL_PASSWORD`: 初回ログイン用パスワード（英字・数字を含む10文字以上を推奨）
- `GEMINI_API_KEY`: Google AI Studioで発行したGemini APIキー
- `LINE_CHANNEL_ACCESS_TOKEN`: LINE DevelopersのMessaging APIチャネルで発行したチャネルアクセストークン

構成はSingaporeリージョン、Freeプラン、CI成功後の自動デプロイです。一定時間アクセスがないと休止し、次のアクセス時に起動まで時間がかかります。公開後は `https://<サービス名>.onrender.com/api/health` を開き、`database`, `gemini_configured`, `line_configured` を確認してください。

Gemini APIキーはGoogle AI StudioのAPI Keys画面で作成します。LINEはMessaging APIチャネルを作成し、チャネルアクセストークンを発行します。LINE通知を受ける各メンバーはボットを友だち追加し、TeamFlowの通知設定で自分のLINE User IDを保存してください。
